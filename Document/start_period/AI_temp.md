# QtChat - 简易网络聊天室

## 项目名称
**QtChat** — 基于 Qt Widgets 的局域网简易聊天室（Client/Server 架构）

---

## 项目需求

1. **服务端程序**：监听指定端口，支持多客户端并发接入。收到消息后先存入数据库，再广播给所有在线客户端。新客户端连接时，从数据库拉取最近 N 条历史消息发送给该客户端（多线程：每个客户端连接一个独立线程处理）

2. **服务端数据库管理**：服务端使用 SQLite 数据库存储聊天记录（表字段：id、时间、发送者昵称、消息内容）。仅限服务端对数据库进行增删改查，客户端不直接访问数据库（数据库）

3. **客户端程序**：通过 TCP Socket 连接服务端，发送消息和接收广播。客户端本地不存储数据库，所有消息数据来自服务端网络传输（Socket 网络编程）

4. **消息列表展示（MVD）**：客户端使用 QListView + 自定义 QAbstractListModel + 自定义 QStyledItemDelegate 展示聊天消息，每条消息显示发送者、时间和内容。Model 数据由网络接收到的消息动态填充（MVD 模型）

5. **基本 UI 布局**：消息列表 + 文本输入框 + 发送按钮 + 在线人数显示，使用 Layout 管理布局（基本组件）

6. **Enter 键快捷发送**：在输入框中按下 Enter 键直接发送消息（键盘事件）

7. **心跳检测**：客户端定时（QTimer）发送心跳包，超时未收到服务端响应则判定断线，UI 上通过信号和槽自动更新连接状态指示（QTimer + 信号和槽）

8. **消息状态图标绘制**：在消息列表中，每条消息前用 QPainter 绘制一个简单的状态小圆点（发送中=黄色 / 已送达=绿色 / 发送失败=红色）（QPainter 绘图）

9. **随机昵称后缀**：客户端首次启动时，使用随机数生成一个 4 位数字后缀拼接到昵称，如 "User#3821"（随机数）

10. **导出聊天记录**：客户端通过菜单栏"文件→导出"，将当前内存中的聊天记录保存为 .txt 文本文件（文件 I/O）

---

## 架构约束
- 数据库仅由服务端管理，客户端不接触数据库
- 客户端通过 MVD 展示消息，Model 数据源为网络消息（内存），非本地数据库

## 注意事项
- 总代码量控制在 **~1500 行**以内（不含注释和空行）
- 服务端 + 客户端为两个独立可执行程序，可共享部分公共代码（如消息协议定义）
- 使用 CMake 或 qmake 管理项目，从简即可
- 聊天为公开聊天室模式（无私聊、无用户注册），保持简洁
- 网络通信仅使用 TCP，暂不涉及 UDP
- 不使用 JSON、QSS、QSettings 等未学过的技术
