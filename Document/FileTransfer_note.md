# 关于高性能文件传输模块的草稿



## 协议

客户端发送文件：

共享文件：协议类型 + 000 + 文件名称 + INTERUPT + 文件大小 + INTERUPT  + 文件内容

私发文件：协议类型 + 001+ 用户名 + INTERUPT + 文件名 + INTERUPT + 文件大小 + INTERUPT  + 文件内容

查询文件：协议类型 + 010 + INTERUPT

接收文件：协议类型 + 011 + 文件名 + INTERUPT



服务器处理

共享文件：协议类型 + 000 + 通知

私发文件：协议类型 + 001 + 通知

查询文件：协议类型 + 010 + 遍历 + 通知

发送文件：协议类型 + 011 + 文件名 + INTERUPT + 文件大小 + INTERUPT + 文件正文





## 客户端接收文件方式

用一个worker
当客户端点击接收文件时，把这个worker移动到子线程
关闭后把worker离开线程

这个worker用来show一个新的窗口(要设计新的ui组件)，里面有`可接收文件列表`，`刷新`，`关闭`，
`可接收文件列表`中要包含如下：

```
文件名(已经有filename + (发送者名称)的形式)
下载文件按钮
```
